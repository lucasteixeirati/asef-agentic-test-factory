# Etapa 6.3.1 — núcleo REST local

**Data:** 2026-07-18

A primeira subfatia cotidiana de API foi implementada após os contratos e o mapa do produto. `api-generate` transforma uma intenção natural em plano tipado por cassette e deixa o plano aguardando revisão. A origem não é escolhida pelo modelo: o runtime injeta a base URL informada pelo operador. `api` revalida o plano, exige a porta na allowlist, executa requests read-only contra endereço literal de loopback e grava relatórios JSON/Markdown.

A política bloqueia hosts externos, redirects implícitos, proxies, headers sensíveis ou de transporte, campos sensíveis em query/body, métodos mutáveis por padrão, respostas excessivas e planos acima do budget. O relatório não persiste corpos de resposta.

A capability foi promovida de `planned` para `partial`, não para `experimental` ou `supported`. Ainda faltam integração ao estado/budget geral da run, execução Docker, provider live com custo reconciliado, OpenAPI e conformance dedicada. A execução atual no host só é aceitável porque o alvo é loopback literal e fictício; não representa autorização para testar serviços públicos.
