# Etapa 6.3.3 — provider, OpenAPI e conformance API

**Data:** 2026-07-18

`api-generate` ganhou modo live opt-in reutilizando o gateway existente, sem chamada real nesta revisão. Modelo, tarifas e budget são explícitos; chamadas, tokens, retries, duração e custo estimado são reconciliados na capability run. Saídas rejeitadas preservam o uso observável para não esconder custo.

OpenAPI 3.0/3.1 JSON passou a limitar o espaço de geração. O loader rejeita chaves duplicadas, referências externas e metadados sensíveis, ignora `servers`, rotas parametrizadas e operações mutáveis, e registra na run somente hash e resumo sanitizado. O documento bruto não é copiado.

O dataset `BACKEND-API-CONFORMANCE-001` declara oito casos executáveis positivos e negativos. A política de rede não foi ampliada: literal loopback evita resolução DNS no executor atual; alvo externo permanece bloqueado até existir HTTPS com resolução e conexão pinadas, isolamento e autorização verificáveis.

Conclusões técnicas foram estruturadas pela IA e permanecem sujeitas à revisão de Lucas. Não houve provider live real, push, tag ou release nesta fatia.
