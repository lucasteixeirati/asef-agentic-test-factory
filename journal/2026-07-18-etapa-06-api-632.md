# Etapa 6.3.2 — run, budgets, evidências e sandbox API

**Data:** 2026-07-18

A intenção natural e a execução REST deixaram de ser comandos sem identidade comum. Um envelope genérico de capability run passou a registrar workflow, skill, perfil, status, classificação, budgets, usage, plano por hash, fatos, histórico e evidências. Ele foi criado separadamente do estado unitário para não introduzir condicionais HTTP no WF-001.

`api-generate` agora cria a run antes da chamada gravada, reserva a chamada, registra tokens e duração, salva o plano e pausa em revisão humana. `api --run-id` retoma a mesma identidade, reconcilia o hash, consome budget de requests e persiste resultado e reports. Adulteração do plano, budget insuficiente, estado incompatível e JSON com chaves duplicadas falham antes do request.

`state.json` e `manifest.json` ganharam escrita em bundle com restauração sob falha injetada. Uma prova Docker real executou fixture e cliente no mesmo container com rede externa desligada. Essa prova não foi convertida em alegação de sandbox para APIs reais: o adapter público ainda roda no host e aceita somente loopback literal.

Não houve chamada de provider live ou custo de API nesta subfatia.
