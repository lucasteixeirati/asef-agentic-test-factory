# Etapa 6.4.1 — contratos, política e fixture Web UI

**Data:** 2026-07-18

Lucas aprovou a continuidade estrita pela 6.4.1. A fatia materializou contratos
framework-neutral de plano e resultado, parser JSON estrito, política `web-ui`,
loader com rejeição de chaves duplicadas e uma fixture estática resetável. A porta
da fixture passou a exigir autorização explícita; paths com origem, fragmento,
backslash ou whitespace são rejeitados; seletores livres e marcadores sensíveis
não entram no plano.

O resultado normalizado reconcilia `PASSED`, `FAILED`, `ERROR`, `TIMEOUT` e
`POLICY_BLOCKED`. Screenshots são aceitos somente como refs PNG contidas em casos
não aprovados e continuam não publicáveis por padrão. O threat model registra que
ainda não existe enforcement real de browser.

Vinte testes locais cobrem contratos, política, adapter, resultado e fixture, com
cobertura branch específica de 87% nos três módulos da fatia.
Playwright, Chromium, compilação TypeScript, execução Docker, provider live Web UI,
push, tag e release não fizeram parte desta fatia. A capability e o perfil Node
permanecem `planned` até as provas posteriores.
