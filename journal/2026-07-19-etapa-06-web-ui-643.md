# Etapa 6.4.3 — compilador e primeira execução Web UI

**Data:** 2026-07-19

Lucas aprovou formalmente a 6.4.2 e autorizou iniciar a 6.4.3. O plano declarativo
agora é compilado para um módulo TypeScript determinístico e sem código livre. O
adapter revalida política, plano, bytes e SHA-256 antes de iniciar o container.

O driver serve a fixture allowlisted em loopback e executa Chromium no mesmo
container com rede desligada. A jornada positiva passou e uma assertion divergente
foi classificada como `FAILED`, gerando PNG contido, privado e não publicável. Os
testes também cobrem adulteração do artifact, origem/porta, resultado spoofado e
política Docker. Uma diferença CRLF do Windows foi detectada pelo hash e corrigida
com escrita LF explícita, sem afrouxar a reconciliação.

A 6.4.3 não cria comando público, provider, capability run, dataset de conformance,
acesso a site externo, publicação ou promoção de suporte. Esses itens continuam
nas fatias 6.4.4 a 6.4.6.
